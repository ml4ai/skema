Module deadpool::unmanaged

Expand description

Unmanaged version of the pool.

“Unmanaged” means that no manager is used to create and recycle objects. Objects either need to be created upfront or by adding them using the Pool::add() or Pool::try_add() methods.

Example

use deadpool::unmanaged::Pool;

struct Computer {}

impl Computer {
    async fn get_answer(&self) -> i32 {
        42
    }
}

#[tokio::main]
async fn main() {
    let pool = Pool::from(vec![
        Computer {},
        Computer {},
    ]);
    let s = pool.get().await.unwrap();
    assert_eq!(s.get_answer().await, 42);
}
